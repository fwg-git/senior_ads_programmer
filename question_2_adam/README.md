# Fanny Gautier interview for Senior ADS Programmer position

## Question 2: ADaM ADSL Dataset Creation

### Objective

Create an `ADSL` (Subject Level) dataset using SDTM source data, the `{admiral}` family of packages, and tidyverse tools.

### Task

Develop an R program (`create_adsl.R`) to create the `ADSL` using the input SDTM data, the `{admiral}` family of packages, and tidyverse tools as explained in the [Pharmaverse examples - ADSL](https://pharmaverse.github.io/examples/adam/adsl.html) or in `{admiral}` [documentation](https://pharmaverse.github.io/admiral/cran-release/articles/adsl.html). Adjust the logic and derive additional variables as mentioned below.

The DM domain is used as the basis of the ADSL. Start by assigning `pharmaversesdtm::dm` to an `adsl` object as explained in [this](https://pharmaverse.github.io/admiral/cran-release/articles/adsl.html#readdata) section of the `ADSL` article.

**Input datasets**: `pharmaversesdtm::dm`, `pharmaversesdtm::vs`, `pharmaversesdtm::ex`, `pharmaversesdtm::ds`, `pharmaversesdtm::ae`.

If time is missing, ie. not collected, then impute completely missing time with 00:00:00, partially missing time with 00 for missing hours, 00 for missing minutes, 00 for missing seconds. **If only seconds are missing then do not populate the imputation flag (TRTSTMF).**                                                                                                                                             |

### Expected Result

-   An error-free program with good documentation that will create the `ADSL` dataset with all the requested variables in the question. These should be derived using {admiral} functions where possible.
-   Resulting ADaM dataset in any format.
-   A text file/log file as evidence for code running error-free (Console saved into a txt file).

### Repository Structure

-   `R` folder contains the scripts.
-   `data` folder contains the resulting and final dataset.
-   `log` folder contains the copy of the console saved as `.txt`.
